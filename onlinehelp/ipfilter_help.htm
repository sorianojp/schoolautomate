IP FILTER HELP

This module allows or blocks one IP or a set of IPs from accessing a module and/or sub-module. If sub-module is not selected or listed, IP filter is applicable to the complete module including its sub-modules by default. There are two types of IP Filter.

(a) Page level IP Filter
      Page level IP Filter implements secured IP Filter to a specific page. IP filter security can only be implemented on the pages with .jsp extension. IP Filter security can't be implemented on html pages or other type of static or active pages. Please contact the vendor for activating ip filter security to specific pages other than .jsp pages.

(b) Module/Sub-module IP Filter
      Module/Sub-module IP Filter is implemented on a complete module or a sub-module that belongs to a module. All the sub-modules under a module is listed when a specific module is selected. If "ALL" option is selected under Module dropdown list, IP Filter is implemented on all the SCHOOLBLIZ modules, and if "ALL" option is selected under Sub-modules dropdown list, IP Filter is implemented on the selected module and all the submodules under it. If there are additional sub-module required to be added under a module, please request the vendor.

There are 4 types of IP Filtering in implementation.

(a) Allow one IP
      The selected module/sub-module is accessible to this IP only. For example if 192.0.0.7 IP is allowed, no client with IP other than 192.0.0.7 can access this module. The IP provied should be a valid IP format (xxx.xxx.xxx.xxx is a valid IP format). This does not accept invalid IP format.

(b) Allow all IP starts With
      The selected module/sub-module is accessible to all the IPs start with the speficied IP. For example if 192.0.0 IP is allowed, 192.0.0.xxx and 192.0.0xx.xxx are the set of IPs allowed. To allow only 192.0.0.xxx IP sets, instead of specifying 192.0.0 , type 192.0.0. (DOT at the end of IP).

(c) Block one IP
      The selected module/sub-module is not accessible to the specified IP. For example if 192.0.0.7 IP is not allowed, 192.0.0.7 is blocked to access the specified module/sub module or a page. The IP provied should be a valid IP format (xxx.xxx.xxx.xxx is a valid IP format). This does not accept invalid IP format.

(d) Block all IPs start with
      The selected module/sub-module is not accessible to all the IPs start with the speficied IP. For example if 192.0.0 IP is blocked, 192.0.0.xxx and 192.0.0xx.xxx are the set of IPs not allowed. To block only 192.0.0.xxx IP sets, instead of specifying 192.0.0 , type 192.0.0. (DOT at the end of IP).
